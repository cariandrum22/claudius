# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.2.1] - 2026-05-04

### Fixed
- Nix flake package metadata now tracks the crate release version so output/store path names match the shipped `claudius` binary version

## [0.2.0] - 2026-05-04

### Added
- Canonical multi-agent skill rendering with `skill.yaml` plus shared `instructions.md` as the portable source of truth
- New skill workflows: `claudius skills validate`, `claudius skills render`, and `claudius skills migrate`
- Canonical per-agent body overrides via `targets/<agent>.md`, alongside existing prepend/append fragments
- Stronger doctor and validation guidance for deprecated full override trees and metadata leakage across agent targets

### Changed
- Skill sync now renders agent-native artifacts for Claude, Claude Code, Codex, and Gemini instead of behaving as a copy-only engine
- Bootstrap templates, documentation, and integration coverage now reflect the canonical skill format and migration path
- Deprecated `skills/<agent>/<skill>/SKILL.md` full overrides remain supported as an escape hatch, but now emit explicit migration guidance

### Fixed
- Codex skill rendering now drops Claude-only metadata by construction while preserving documented Codex-specific overlays
- Deprecated full override trees can now be migrated into canonical target overlays without losing agent-specific instructions

## [0.1.2] - 2026-04-30

### Added
- `claudius config doctor` to audit managed, best-effort, legacy, and unmanaged configuration surfaces
- Agent-specific asset sync and validation for Claude Code, Codex, and Gemini, including Gemini system/system-defaults targets and Codex admin TOML targets
- Configurable 1Password auth modes for `claudius secrets run`, including service-account support for headless environments

### Changed
- Skills sync now merges shared skills, agent-specific overrides, and legacy `commands/*.md` sources with safe prune behavior
- Codex skills now default to the official `.agents/skills` target; `.codex/skills` is now an optional compatibility copy and `--enable-codex-skills` is deprecated
- CI workflows, Nix inputs, Rust toolchain metadata, and dependency baselines were refreshed to current supported versions

### Fixed
- Pruning only removes Claudius-managed synced assets instead of touching unrelated files in target directories
- Validation, bootstrap output, and documentation were aligned with the current managed agent surfaces and 1Password configuration model

## [0.1.1] - 2026-01-12

### Added
- Codex: optional global sync for admin files `requirements.toml` (`--codex-requirements`) and `managed_config.toml` (`--codex-managed-config`)
- Gemini: optional global sync to the system settings file (`--gemini-system`)
- `codex.managed_config.toml` template + validation support

### Changed
- Context files standardized on `CLAUDE.md` (Claude/Claude Code) and `AGENTS.md` (Codex/Gemini)

### Fixed
- macOS: reduce repeated Touch ID prompts during `claudius secrets run` when resolving `op://...` secrets via 1Password CLI

## [0.1.0] - 2025-07-23

### Added

#### Core Features
- Multi-agent configuration management for Claude, Codex, and Gemini
- MCP (Model Context Protocol) server synchronization
- Project-local and global configuration modes
- Context file templates (CLAUDE.md, CODEX.md, GEMINI.md, AGENTS.md)
- Custom slash command management
- Rule-based context file generation

#### Secret Management
- 1Password CLI integration with op:// reference support
- DAG-based variable expansion for nested environment variables
- Circular dependency detection
- Secure command execution with automatic secret resolution
- Support for both bare and delimited op:// syntax

#### Commands
- `init` - Bootstrap configuration with sensible defaults
- `sync` - Synchronize configurations to target files
- `append-context` - Add rules to context files
- `install-context` - Install rules with reference directives
- `run` - Execute commands with resolved secrets

#### Developer Experience
- Comprehensive test coverage (150+ tests)
- Nix flake support for reproducible builds
- Pre-commit hooks for code quality
- Just command runner integration
- Linux and macOS support (Unix-like systems)

#### Configuration
- TOML-based application configuration
- JSON support for MCP servers and settings
- Skills and rules support
- XDG Base Directory compliance

### Notes
- Initial release developed as an AI-agent-driven development experiment
- Significant portions of the codebase generated by Claude Code
- No copyright claimed on AI-generated content
